# フェーズ5: 検証と整形

## 目的

実装後のコンパイル、整形、チュートリアル動作確認、通常バトル回帰確認を行う。

## 実施手順

### 1. コンパイル

- `mcp__uLoopMCP__compile` を `ForceRecompile=false` で実行する
- コンパイル成功を確認する

### 2. エラーログ確認

- `mcp__uLoopMCP__get-logs` を `LogType=Error` で実行する
- エラーが残っていないことを確認する

### 3. フォーマット適用

- `dotnet format whitespace FormatCheck.csproj --verbosity diagnostic`
- `dotnet format style FormatCheck.csproj --severity warn --verbosity diagnostic`
- `dotnet format analyzers FormatCheck.csproj --severity warn --verbosity diagnostic`

### 4. チュートリアル動作確認

- 入札開始直後に全カードと確定ボタンが無効であることを確認する
- Joy を選ぶまでカード選択へ進まないことを確認する
- 指定カード以外を選べないことを確認する
- 必要量に達するまで増加でき、到達後は増加不可になることを確認する
- 確定後に競合フェーズまたは後続フローへ正しく遷移することを確認する
- デッキ選択で指定カードのみドラッグできることを確認する
- カードバトルでコインフリップ固定が反映されることを確認する
- 指定カードのみ配置可能なラウンドが正しく動くことを確認する
- 指定ラウンドでスキル表示と使用が制御されることを確認する
- 毎回同じ展開になることを確認する

### 5. 回帰確認

- 通常バトルで入札 UI とカードバトル UI が従来通り動作することを確認する
- 通常バトルにチュートリアル制約が漏れていないことを確認する

## 完了条件

- コンパイルエラーがない
- フォーマット適用が完了している
- `alv` 戦のチュートリアル進行が成立する
- 通常バトルの主要フローに回帰がない

## 注意点

- チュートリアルだけでなく通常戦の確認を必ず行う
- 問題が出た場合は、どのフェーズの変更で崩れたかを切り分けて戻る
